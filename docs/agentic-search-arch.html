Core Agentic Systems

Preflight
Intent + query steering
Combines heuristic detection with a short-timeout fail-open planner LLM call to improve first-tool selection.
Tool Selection
Risk-aware metadata
Tool execution meta controls batching and concurrency-safe parallel execution for read-only operations.
Skills Modularity
Shared orchestration + runtime modules
Web/search/network logic is isolated in modules/web-runtime.js, while shared.js focuses on orchestration and runtime wiring.
Context Manager
Budget, microcompact, summarize
Large tool outputs are compacted before storing; older tool results are cleared; summarize runs only when required.
Loop Guardrails
Semantic anti-repeat
Near-duplicate web_search signatures are blocked to avoid repetitive loops across rounds.
Prompt Safety
Injection-aware continuation
Tool outputs are treated as untrusted input; suspicious payloads are tagged and converted into runtime continuation safety reminders.
Memory
Long-term + session persistence
Sessions, runtime cache, long-term memories, preferences, tasks, and todos are persisted in localStorage.
Query Strategy
Bounded recursive orchestration
Each round decides: call tools, continue with evidence, compact context, or force final answer at max rounds.
Snapshot Port
Dist reuse + adapted JS runtime
Imported snapshot skill metadata and sanitized prompt snippets are reused directly in runtime prompt/tool orchestration.

Query Strategy and Transition Rules

  • Round-bounded loop with deterministic stop conditions.
  • If model output has no usable tool call and no final text, the loop asks for a corrective continuation.
  • Repeated failures disable exact tool signatures; semantic signatures disable near-duplicate web_search calls.
  • Structured continuation blocks are injected for tool summaries, permission denials, and compaction notes.
  • Suspicious prompt-injection patterns in tool results trigger explicit safety reminders before the next model turn.
  • Context manager runs after each tool execution phase to keep prompt growth controlled before hard-limit summarization.
  • When max rounds are reached, the model is forced to produce a final evidence-based answer.

Implementation Anchors

  • src/app/agent.js: loop state, tool execution, context manager pipeline, guardrails, and final-answer forcing.
  • src/core/orchestrator.js: sectioned system prompt builder and runtime continuation reminder composer.
  • src/app/runtime-memory.js: long-term memory manager and multi-scope retention cache.
  • src/skills/snapshot-adapter.js: sanitized snapshot prompt/skill bridge into runtime.
  • scripts/build-snapshot.mjs: snapshot transpile + prompt snippet extraction pipeline.
  • src/skills/shared.js: preflight plan, optional planner LLM call, runtime orchestration, and registry wiring.
  • src/skills/core/*: intent detection and tool execution metadata.
  • src/skills/modules/web-runtime.js: search providers, web fetch pipeline, weather/geolocation, and page utilities.
  • src/skills/modules/*: filesystem/data/registry runtime responsibilities.
  • src/skills/groups/*.js: flattened UI group descriptors for tool families.
